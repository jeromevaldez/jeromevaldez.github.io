Lessons learned (possibly out of date)

  • Agents love writing code
    • So sometimes I have asked it to look at what it's building and ask if it can solve the problem by simplifying instead of writing more.
  • Agents want to please you soooo badly
  • Agents don't always 'see' UI/UX very well
    • This means you can paste screenshots but my results have varied
    • Some tools use Figma connectors, but those didn't impress me when I tried. I needed to have really clean figma files, with great layer names.
  • Agents know surprisingly little about themselves and the products in which they live
  • Patience is important
  • Grammar is not important ðŸ’¡
  • Sometimes I feel like I'm a copy/paste monkey
  • Don't fear the undo, even deep into a conversation ðŸ’¡
    • Asking the agents to undo isn't as clean as clicking 'Revert'
  • GitHub version control helps with the fear or losing work
    • Do small commits (this is good dev practice) vs doing a ton of changes in one commit. It's easier to find and easier to roll back to a specific branch/feature ðŸ’¡
  • Use multiple monitors. Use all the pixels. Use multiple windows.
  • I prefer tools that work in my local files and folders
    • Because I can point to other folders with relative paths and it can 'read' those other projects and bring those designs into the current project ðŸ’¡
    • Because it feels faster
    • Because there are other tools I can use locally to manipulate those files and folders
  • Don't forget to think about how to share your UX prototypes. e.g. included deployments in the tools, or Vercel, or Netlify, etc. ðŸ’¡

9/14

Use multiple tools and multiple models

  • Different models are better at some things
  • Sometimes using the same model in a different UI/context actually works
  • Sometimes I have ChatGPT open to help me with using Cursor, which uses Claude 4 Sonnet or Gemini 2.5 Pro
  • I have used ChatGPT to help me think through an idea and build a rather large prompt or a requirements document as a markdown file. I then put that .md file into my project folder and ask the agent to build according to that file. ðŸ’¡
10/14

Experiment with as much as you can

  • Free tiers are pretty generous
  • Paying for a month or the first set of credits is very enlightening and doesn't have to be permanent. I have tried and quit several subscriptions.
  • Maybe consider keeping track of all the tools and which logins you used and which ones you are still paying for or have cancelled ðŸ’¡
    • Sometimes I login with Google
    • I login with Github when the product allows it because I am more likely to try out version control with the product if I stay on it for a while
  • Nobody really knows this stuff. So just dive in. Learn. Maybe even teach. And have fun.
