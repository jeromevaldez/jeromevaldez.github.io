Javascript

What is javascript?

It was invented in 1995 by Brendan Eich at Netscape (an early web browser). It’s the programming language for the web, written specifically for the web. Remember the three-legged stool? This leg defines behaviours and interactivity of web pages. Which behaviours you might ask? Here are some examples (but there are many many more:

  • onMouseOver behaviours like a hover
  • What happens when a form is filled out incorrectly?
  • How long does a carousel autoplay?

What it's not (or, how it differs from the back end)

Javascript is still considered part of the front-end of a website as opposed to back-end or server-side code. Javascript is a client-side language. The client (in our case the web browser) runs the code that's sent by the server. Languages like PHP or Java or .Net or Ruby on Rails are all server-side languages - they run on the server. Even though it's very rare, a user can disable javascript in the browser and it will prevent it from running.

Javascript is not Java
It’s not at all related to Java. Sun’s Java language was really hot at the time of Javascript’s invention. So it instead of naming it LiveScript, they named it Javascript instead in order to take advantage of the popularity of Java at the time (1995).

All that said, it is still a programming language and uses concepts that should be familiar to those of you who know programming: arrays, loops, strings, variables, etc.

DOM (the Document Object Model)

The document object model is one of the most important things to understand if you are going to work with javascript. Let's break down what the DOM means.

D is for Document

This is the web page.

O is for Object

Object are simply the things (duh!). These are elements and components of the document. This includes not only the tags like h1, ul, but also the content within, and even the attributes inside them.

M is for Model

The Model is the representation of the document as a tree structure, a hierarchy, with parents and children and siblings. It’s a set of terms and standards that describe how to interact with pieces of the web page.

Events

One of the other important things to understand are events. Events are things that happen on the webpage - they are already happening.

  • Pages loading
  • Users moving their mouse
  • Users clicking their mouse
  • Users pressing their keyboards
  • Users giving focus (moving the cursor) to a form field

Events are always happening. You don’t write events. You write the event listener or handler. In other words you respond to the events that happen.
