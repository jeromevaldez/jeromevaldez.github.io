Web fonts - what are they and how do I use them?

"Unlike web safe fonts, web fonts are not pre-installed on the user's system. The fonts are downloaded by the user's browser while rendering the webpage, and then applied to your text. The main drawbacks of using web fonts is it will slow your site's load time ... If a browser is unable to use the web font it will fall back on the web safe fonts in the stack." - from cssfontstack.com

How to use Web Fonts on your sites

Step 1: Pick a Web Font

You should go find a font that you want to use. There are many freely available ones on Google Fonts. There are also several web fonts that you need to pay a license fee to use. The most popular are hosted by services such as Fontdeck, TypeKit, Fonts.com, and Hoefler & Co.â€™s Cloud.typography.

Step 2: Add some stylesheet link to the html pages on which you want to use the font

You'll insert a snippet like the below within the <head> tags of your webpage

<link href='https://fonts.googleapis.com/css?family=Roboto' rel='stylesheet' type='text/css'>

Step 3: Apply the font in your css file

The css rule below is saying that every h1 element should use the Robot font, but if that's not available for some reason, then fall back to use Arial, and if that doesn't work, fall back to use the system's default sans-serif font. 


h1 {
	font-family: 'Roboto', 'Arial', 'sans-serif';
}

Images on the web

How large can my images be?

A good rule of thumb for e-commerce sites is to keep product images under 70kb. If you're working on other types of sites, you might be able to get away with larger. Your portfolio sites might have images in the 200kb range. Remember though, smaller is faster and faster is better when it comes to digital. And by smaller I mean file size not the visual scale on the page.

How can I shrink my images, but still maintain their quality?

The tools within your Adobe products include saving for the web. You'll likely use Photoshop to do this with your design and illustration portfolio projects. I am assuming that your other courses have covered the steps to do this already. You can experiment with the quality and size settings to get an image that is small, but of acceptable quality.

There are some online services that can take one of your large images and optimize it.

  • Kraken.io
  • JPEG Mini
  • Smush.it
  • Tiny PNG

Inserting background images using HTML and CSS

You'll sometimes want to place an image as a background instead of inside the flow of the document. Here's how:

First you'll need to define the html block that will contain the background image. This usually is a div, but you can do this to most html block-level tags. Then you'll apply css rules to that block. The best way (in general) to do this is by giving the block a class, like so <div class="bgimage">some content goes here</div>

Now you'll create a css rule that applies your chosen background image to the block.

                	
.bgimage {
	background-image: url("img/pumpkins.jpg");
}

Within the same rule, you can make other declarations such as:

background-repeat: no-repeat or repeat-x or repeat-y; this will cause your background image to not repeat, or repeat horizontally, or repeat vertically.

background-position: right top; this will cause your background image to be positioned relative to the box in which it's contained. In the rule I just wrote, it will be positioned at the top right of the block in which it's contained. The different ways you can state background-position can be found on this page.
