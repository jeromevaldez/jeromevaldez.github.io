Project 2: Information Architecture

Due: March 21st at the start of class 9am.

Use Adobe Muse to create two different sites. The first will reflect the information architecture of the current website. The second will show what you recommend as a new information architecture. You'll use the same website as the subject for both.

One example website in class was The Old Spaghetti Factory. Other possible candidate website for this project were written in Slack. If you have an alternative restaurant website, please run it by me for approval first. The website should have between 20-30 pages minimum and should have some depth to the information architecture. In other words, there should be some clear child-parent relationships between the pages. It should go at least two levels deep and it can go deeper than that.

The Muse pages should include a navigation/menu bar. The pages do not have to be completely designed with full content on them, but they should have some text or title on each page that shows which page is being viewed. We did this in our class exercise when we wrote "This is the Product page" or "This is the Beer Archive page".

Your second Muse site showing the recommended information architecture should still be able to incorporate all of the content of the original site. You can reorganize it and combine content onto one page, but you should not be removing any content that was on the original.

What I'll be looking for:

  • Your assignment is handed in as a URL. You'll point to a server that you've set up on your host. For example http://yourdomain.com/ides151/project2/
  • The current information architecture reflects the real site and is not missing any key content. Watch out for sneaky content that is not obvious or in the main navigation. This includes Privacy pages, Terms and Conditions, and even Thank You pages for websites that have forms that users submit.
  • The new recommendations that you put forward are an improvement to the information architecture. The recommendations should not make it harder to find content or more confusing to the users.
  • If you make changes to combine or add content, then you should provide a written rationale about why you made those changes. You can submit this in a simple written document.
  • Your new recommended information architecture is not missing any content that the original site had.
